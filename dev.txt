# Feature: Voice Agent Bug Tracker & Call Quality Tagging

📝 **Description**

This ticket implements two interconnected features that enable quality control for voice agent calls: a voice agent bug tracker (with configurable bug type taxonomy) and call quality tagging (which replaces the bookmark button). Together, they provide superadmin-configurable bug type options and reviewer-facing tools to tag and filter calls by quality outcome and specific bug types observed.

**What:**

* Database: New `super_configs` table — generic configurable option taxonomy with category (e.g., `voice_bug_type`), name, display_name, description, options JSONB, is_active, sort_order, timestamps, and unique constraint on (category, name).
* Seeded 6 default voice bug types: Transcription Error, Agent Stopped Talking, Incorrect Escalation, Duplicate Response, Script Deviation, Call Drop.
* `evals.bug_type_ids` JSONB column: stores list of super_config IDs tagged on an eval during review.
* `raw_calls.quality_tag` VARCHAR(30): stores AGENT_HANDLED_WELL or AGENT_FAILED (set via popover picker in eval form).
* `raw_calls.quality_tag_notes` TEXT: optional notes explaining the quality tag.
* `raw_calls.is_bookmarked` kept in sync: automatically set true when quality_tag is set, false when cleared.
* Backend API endpoints:
  - GET `/api/v1/super-configs?category=voice_bug_type` — list active configs (reviewers), all configs (admins/superadmin).
  - POST `/api/v1/super-configs` — create new config (superadmin only).
  - PATCH `/api/v1/super-configs/{config_id}` — update config (superadmin only).
  - GET `/api/v1/calls/{call_id}/bug-type-ids` — retrieve resolved bug types tagged on a call's eval.
  - PATCH `/api/v1/calls/{call_id}/bug-type-ids` — replace bug_type_ids list on a call's eval.
  - PATCH `/api/v1/calls/{call_id}/quality-tag` — set or clear quality tag and sync is_bookmarked.
* Frontend: New `BugTypeConfigPage.tsx` (superadmin page moved into TeamPage as tab) to CRUD bug types with severity stored in options JSONB.
* EvalFormPage: Bookmark button replaced with quality tag picker popover (two radio options: "Handled Well", "Agent Failed"); stored locally until eval submit.
* CallDetailPage: Quality tag badge displayed as green/red pill near bookmark icon in right panel.
* AllCallsPage: Bookmarked filter + nested Quality Tag sub-filter (only visible when Bookmarked=true); wired to backend via quality_tag query param.
* BugsPage: New filter bar redesigned with flat structure: BugType selector + 7-day/30-day toggle; severity filter resolves super_config.options->>'severity'.
* DashboardPage: Bug Occurrences card shows top 5 bug types by count; 7-day/30-day time range toggle.
* Filter Bar Redesign (CallFilterBar & BugsFilterBar): Collapsed into Search + Date + Filters(n) popover + Sort; popover contains segmented pill groups per category.
* BookmarkedCallsPage: Deleted; `/bookmarked` route redirects to `/all-calls`.
* CallDistributionPage & BugTypeConfigPage: Moved into TeamPage as tabs.
* SUPERADMIN_NAV sidebar items: Removed; accessed via Team tabs.
* Backend: `quality_tag` filter added to `/api/v1/calls/list`, `/api/v1/calls/count`, `/api/v1/calls/stats` endpoints.
* Backend: `severity` filter added to `/api/v1/bugs/list`, `/api/v1/bugs/count` endpoints; resolves via super_configs subquery.

**Why:**

* Voice agent quality monitoring requires domain-specific bug taxonomy that can be extended by superadmin without code changes.
* Reviewers need a faster, more intuitive way to flag high-quality vs. failed calls (replaces bookmark + separate bug form).
* Severity filtering on bugs enables triage by business impact (low/medium/high stored in super_config.options).
* Quality tags sync with bookmarks to preserve existing workflows while adding richer semantics.
* Consolidated team config (call distribution, bug types) into TeamPage reduces sidebar clutter and simplifies navigation.

This enables the product to capture structured quality feedback that drives continuous improvement of the voice agent, supports data-driven triage, and allows superadmins to adapt bug types to evolving agent capabilities without development effort.

✅ **Acceptance Criteria**

### Database & Migrations
* Alembic migration `j8k2l3m4n5o6_add_super_configs_and_voice_bug_tagging.py` successfully creates `super_configs` table with columns: id, category, name, display_name, description, options (JSONB), is_active, sort_order, created_at, updated_at.
* Unique constraint `uq_super_configs_category_name` on (category, name) prevents duplicate bug types.
* Index `idx_super_configs_category_active` on (category, is_active) exists for query performance.
* Migration seeds 6 default voice_bug_type configs with sort_order 0–5.
* `evals.bug_type_ids` JSONB column added and nullable.
* `raw_calls.quality_tag` VARCHAR(30) column added and nullable.
* `raw_calls.quality_tag_notes` TEXT column added and nullable.

### API: SuperConfigs
* GET `/api/v1/super-configs?category=voice_bug_type` returns 200 with list of SuperConfigRead objects.
* Reviewers receive only active configs; admins/superadmin receive all configs (filtered via `active_only` flag).
* POST `/api/v1/super-configs` with SuperConfigCreate payload returns 201 with created config (superadmin only); returns 409 if duplicate (category, name).
* PATCH `/api/v1/super-configs/{config_id}` returns 200 with updated config or 404 if not found (superadmin only).

### API: Call Tags & Quality Tags
* PATCH `/api/v1/calls/{call_id}/quality-tag` with valid quality_tag (AGENT_HANDLED_WELL or AGENT_FAILED or null) returns 200 with updated RawCall.
* Setting quality_tag automatically sets is_bookmarked=true; clearing (null) sets is_bookmarked=false.
* Invalid quality_tag values (not in VALID_QUALITY_TAGS) return 404.
* GET `/api/v1/calls/{call_id}/bug-type-ids` returns 200 with list of BugTypeWithName objects resolved from super_configs.
* PATCH `/api/v1/calls/{call_id}/bug-type-ids` with BugTypeIdsUpdate (list of ints) returns 200.

### API: Filtering
* GET `/api/v1/calls/list?quality_tag=AGENT_HANDLED_WELL` returns only calls with matching quality_tag.
* GET `/api/v1/bugs/list?severity=high` filters bugs via subquery: inner join evals on bug_report.eval_id, left join super_configs on evals.bug_type_ids @> super_config.id, filter where super_config.options->>'severity' = 'high'.
* `/api/v1/calls/count?quality_tag=AGENT_FAILED` returns correct count.
* `/api/v1/calls/stats?quality_tag=AGENT_HANDLED_WELL` returns stats for filtered calls.

### Frontend: Bug Type Config
* Superadmins can access BugTypeConfigPage via TeamPage → "Bug Type Config" tab.
* Page displays table of all super_configs in voice_bug_type category (active + inactive).
* Create form: inputs for name, display_name, description, severity (low/medium/high in options JSONB), is_active toggle.
* Edit modal: pre-populate all fields, PATCH to `/api/v1/super-configs/{id}`.
* Delete: soft-delete via PATCH with is_active=false.
* Severity options stored in super_config.options as { severity: "low" | "medium" | "high" }.

### Frontend: Quality Tag Picker
* EvalFormPage: Bookmark button replaced with quality tag selector (popover with radio options: "Handled Well", "Agent Failed", clear).
* Stored locally in `localQualityTag` state until eval form submit.
* Eval submit endpoint includes quality_tag in payload.
* CallDetailPage: Quality tag badge displayed as green pill (AGENT_HANDLED_WELL) or red pill (AGENT_FAILED) near bookmark icon in right panel.

### Frontend: Call Filtering
* AllCallsPage: "Bookmarked" filter + "Quality Tag" sub-filter (appears only when Bookmarked=true); sub-filter shows radio/checkbox options: All, Handled Well, Agent Failed.
* Selected quality_tag appended to query: `?is_bookmarked=true&quality_tag=AGENT_HANDLED_WELL`.
* Filters persisted in URL and restored on page load.

### Frontend: Bug Filtering
* BugsPage filter bar: BugType multi-select, 7d/30d time range toggle, Severity sub-filter (low/medium/high).
* Selected filters appended to query: `?bug_type=id1,id2&days=7&severity=high`.
* Dashboard: Bug Occurrences card lists top 5 bug types by count; 7d/30d toggle switches time range.

### Frontend: Navigation
* BookmarkedCallsPage deleted; accessing `/bookmarked` redirects to `/all-calls`.
* CallDistributionPage and BugTypeConfigPage accessible via TeamPage tabs (not sidebar).
* SUPERADMIN_NAV items removed from AppSidebar.
